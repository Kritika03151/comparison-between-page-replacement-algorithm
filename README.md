# FIFO vs LRU Page Replacement Algorithm Comparison

## Project Overview

This project is a web-based Operating System simulator that compares two important Page Replacement Algorithms:

- FIFO (First In First Out)
- LRU (Least Recently Used)

The application allows users to:
- Enter a page reference string
- Specify the number of frames
- Compare page faults generated by FIFO and LRU algorithms
- Visualize results using a bar chart

The project is developed using:
- Python Flask
- HTML
- CSS
- Matplotlib

# Algorithms Implemented

## FIFO (First In First Out)

FIFO replaces the oldest page in memory when a new page needs to be loaded.

### Advantages
- Simple implementation
- Low overhead

### Disadvantages
- Can suffer from Belady’s Anomaly

---

## LRU (Least Recently Used)

LRU replaces the page that has not been used for the longest time.

### Advantages
- Better performance in most cases
- Uses recent usage patterns

### Disadvantages
- Slightly more complex implementation

# How It Works

1. User enters page reference string
2. User enters frame capacity
3. Application calculates:
   - FIFO page faults
   - LRU page faults
4. Results are displayed
5. Bar chart comparison is generated dynamically
